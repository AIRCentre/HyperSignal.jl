# Road to v1.0

## Lessons from prior art

- **Attribute serialization is never "done".** HypertextLiteral still has an open spec-conformance bug five years in (`<`/`>` in attribute values, https://github.com/JuliaPluto/HypertextLiteral.jl/issues/42), and Hyperscript never resolved boolean-attribute semantics (https://github.com/JuliaWeb/Hyperscript.jl/issues/20). Pin every attribute-escape rule with a doctest *and* a `text/html` round-trip test now, before users encode workarounds for whatever HyperSignal gets wrong.
- **Raw / unescaped passthrough will be requested — design it now or get it bolted on later.** Hyperscript shipped without it, then took years to land an idiom (https://github.com/JuliaWeb/Hyperscript.jl/issues/14). Same with HypertextLiteral and "interpolate a tag string" (https://github.com/JuliaPluto/HypertextLiteral.jl/issues/41). Publish *one* documented escape hatch (e.g. `SafeHTML(s)`) with a security warning, so the third-party version doesn't appear.
- **Type piracy on `Vector{Node}`/`length`/`iterate` is a footgun that breaks downstream on Julia version bumps.** Tim Holy hit it in Hyperscript on 1.6 (https://github.com/JuliaWeb/Hyperscript.jl/issues/24); HypertextLiteral broke on 1.10 via `hasmethod` in `@generated` (https://github.com/JuliaPluto/HypertextLiteral.jl/issues/28, https://github.com/JuliaPluto/HypertextLiteral.jl/issues/33). Audit HyperSignal for `Base.length`/`iterate`/`show` on types we don't fully own, and forbid `@generated` over `hasmethod`.
- **MIME and `show` rules drift between Pluto / IJulia / REPL / HTTP.jl.** Hyperscript bug #22 (HTTP.Response interop), HTL bug #11 (PrettyTables-in-Pluto), HTL bug #10 (clipboard). Add a CI matrix entry that renders one fixture through `text/html`, `text/plain`, and an HTTP.jl response, and asserts the bytes.
- **The "1.0" question becomes a community ask.** Hyperscript got pinged five years in (https://github.com/JuliaWeb/Hyperscript.jl/issues/46); HTL just shipped 1.0 in Jan 2026 after sitting at 0.9.x for ~3 years (https://github.com/JuliaPluto/HypertextLiteral.jl/issues/44). Define the 1.0 criteria publicly so we can ship when met instead of negotiating.

## Pre-v1.0 milestones

- Lock the public surface — move app-grade helpers (`signal_dialog`, `help_tooltip`, `preset_button`, etc.) behind a `HyperSignal.Helpers` submodule; only re-export the rendering core from the top level.
- Document the *one* unsafe-passthrough idiom (`SafeHTML` or equivalent) with a "you own the escaping" warning and a security-review doctest.
- Add an HTML5 attribute-escape conformance suite (covering `<`, `>`, `&`, `"`, `'`, NUL, surrogate pairs, unpaired UTF-16) cross-checked against `EzXML`/browser behavior; pin as doctests.
- Add a boolean-attribute policy (`checked=true` → bare attr; `checked=false` → omit) with a doctest, since both prior libs got this wrong.
- Add a MIME round-trip test (`text/html` ↔ `text/plain` ↔ HTTP.jl `Response` body) for one canonical fragment.
- Add a Pluto-render smoke test (load HypertextLiteral + HyperSignal in a Pluto cell via CI, assert output bytes).
- Audit and remove any `Base.length`/`iterate`/`show`/`==` methods on `Vector{<:Node}` or similar non-owned containers; document the policy in `CONVENTIONS.md`.
- Forbid `@generated` + `hasmethod` in the codebase (CI grep); pick dispatch instead.
- Publish a benchmark table in the docs (small fragment, large table, streaming-IO) anchoring the ~290 ns claim with versioned numbers.
- Mark any v0.x experiment we already regret with `Base.depwarn` in 0.2/0.3, so the deprecation window precedes 1.0.
- Land one external dependent (an internal app, or a Pluto demo notebook in the docs) — closes the "no real downstream users" 1.5-point gap from the 7.5/10 review.

## v1.0 release criteria

- [ ] Public surface frozen and documented; `Helpers` submodule split landed.
- [ ] HTML5 escape suite + boolean-attribute policy + MIME round-trip green on CI (Julia LTS + stable + nightly).
- [ ] Datastar typed-action layer locked, with one signed example app in the wild.
- [ ] Zero open `bug` issues older than 30 days; all known footguns either fixed or in CHANGELOG as "won't fix, here's the idiom".
- [ ] CHANGELOG has an explicit "API stability" section listing what 1.0 promises (and what stays internal).
- [ ] Benchmark table published, regenerated by `make bench`, with regression CI threshold.
- [ ] Security review: attr/tag-name validation, escape codeunit fast-path, and unsafe-passthrough each have an adversarial doctest.
- [ ] At least one downstream user (internal app or registered package) on the released 0.x.

## Out of scope for v1.0

- Pretty-printing / structural `show` for Nodes (Hyperscript #28 — defer; cosmetic, easy to add in 1.x without breakage).
- A templating-language frontend a la `@htl` string macro (Hyperscript #21, HTL's whole design) — HyperSignal's wedge is the function/tag API; a macro is a 1.x experiment, not a 1.0 commitment.
- CSS-in-Julia / scoped-styles DSL (HTL #25, Hyperscript hyphen-CSS #40) — deliberately not our problem; document the workaround.
- Full HTML spec conformance for exotic constructs (`<!doctype>` as a first-class node, comments, CDATA) — HTL #21 / Hyperscript #44; provide raw-passthrough and move on.
